# 注释检查 — 完整性 · 准确性 · 可读性

请按以下流程逐步检查当前项目的代码注释质量，最后给出一份中文报告。

---

## 检查标准

本项目最佳实践：
- JavaScript/Vue 代码注释用中文，关键函数写 JSDoc（`/** ... */`）
- Vue 组件 `<script setup>` 顶部应有组件用途说明
- Express 路由处理函数应标注功能和参数
- 目标：**关键函数和复杂逻辑必须有注释**，不强制行数比例，但核心模块注释覆盖率高

---

## 第 1 步：确定检查范围

先问用户要检查哪些文件。如果用户没指定，默认检查最近修改过的源码文件。

只检查源码文件：`.js`、`.vue` 等，跳过 `node_modules/`、`dist/`、`.claude/` 等。

---

## 第 2 步：逐文件检查

### 维度 A：注释覆盖率（完整性）

对 Vue 组件：
- 组件 `<script setup>` 顶部是否有说明？
- 复杂 props 是否有注释？
- Pinia store 中的 action 是否有说明？

对 Node.js/Express 文件：
- 每个路由处理函数是否有说明？
- 服务层（service）的导出函数是否有 JSDoc？
- 关键逻辑（LangChain 调用、SSE 流式处理、数据库查询）是否有行内注释？

打分：⭐（严重不足）← → ⭐⭐⭐⭐⭐（完全达标）

### 维度 B：注释准确性

检查注释是否和代码逻辑一致：
- 函数名和注释描述的功能是否匹配？
- 参数注释和实际参数是否一致？
- 过时的注释（如功能已改但注释未更新）→ 标记为 ⚠️

### 维度 C：小白可读性

- 新手能通过注释理解代码在做什么吗？
- 专业术语（LangChain、SSE、Embedding、RAG、Chroma、JWT）有简要解释吗？
- 复杂业务流程（旅游推荐流程、知识库检索流程）是否有流程说明？
